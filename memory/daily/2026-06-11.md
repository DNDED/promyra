## Phase 3: v0.6.0 brainstorm + design (this turn)
- **Trigger:** Sid: "ok whats next" (after v0.5.0 was shipped but uncommitted).
- **Decision:** v0.6.0 Agent Swarm v1 (the next roadmap release).
- **Brainstorm answers (in order):**
  1. Scope = Full swarm (orchestrator + planner + builder + critic + test-runner + researcher + worktrees + verification + cost caps + retry)
  2. Autonomy = Fully autonomous, plan on demand
  3. Worktrees = Per-role worktrees, isolated writes
  4. Failure = Retry-with-feedback (2 retries default), then escalate
  5. Cost cap = Soft warn at 50%, hard kill at 100%, $2.00 default
  6. Model routing = Both (subagent pinning + per-tool cascade)
- **Spec approved:** `docs/superpowers/specs/2026-06-11-pi-pro-v0.6.0-design.md` to be written this turn.
- **Plan approved:** `docs/superpowers/plans/2026-06-11-pi-pro-v0.6.0.md` to be written this turn.
- **Key design choices:**
  - File-based scratchpad at `.pi-pro/swarm/<id>/` (durable, diffable, survives restarts)
  - 3-stage parallel-friendly topology: PLAN → RESEARCH+PLANNER (parallel) → BUILD → TEST → CRITIQUE → DECIDE → MERGE
  - 5 subagents: planner, researcher, builder, critic, test-runner
  - 2 cheap (planner, researcher, critic, test-runner), 1 main (builder)
  - Multica preserved as one-shot direct dispatch (bypasses orchestrator)
  - Per-role worktrees under `.pi-pro/worktrees/<id>/<role>/`
  - Verification gate = test-runner exit code; retry builder on fail
  - Cost telemetry in new TUI `<SwarmPanel>` component
