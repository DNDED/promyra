## Bugs hit + fixed this turn (8)

1. `wordEndN` from cursor on word: was advancing to end of next word; now advances to end of current word
2. `wordEndN` at end of word: was staying; now advances to end of next word
3. `applyLineChange` on first line: was deleting line+`\n`; now keeps trailing `\n` (cc semantics)
4. `applyLineDelete` last line: kept `text.slice(0, lineStart) + text.slice(lineEnd + 1)` which left trailing `\`n`; now correctly removes preceding `\`n`
5. `applyLineChange` on last line: kept `text.slice(0, lineStart) + text.slice(lineEnd)` which kept `\`n`; now removes preceding `\`n`
6. `withUndo(runtime, mutate)`: was passing `next` (mutated state) to `pushUndo`, snapshotting the NEW text; now snapshots `oldState` before mutation
7. `setCursor`: was setting `anchor: c` (new cursor) on every call, breaking visual mode anchor tracking; now leaves anchor alone
8. `pushUndo` undoSnapshot bug: was capturing current state instead of pre-mutation; fixed via withUndo arg ordering
