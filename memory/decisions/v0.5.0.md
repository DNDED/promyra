# Decisions — pi-pro v0.5.0

## Token/Cost Foundation release

### Cost math: cache writes REPLACE input tokens, not additive
- **Why:** First test of `estimateCost` for cache write uplift failed (`expected 3.075, received 3.375`).
- **Original (wrong):** `cacheReadBillable = inTokens - cacheRead; usd = cacheReadBillable * input + outTokens * output + cacheRead * cacheRead + cacheWrite * cacheWrite`.
- **Correct:** `regular = inTokens - cacheRead - cacheWrite; usd = regular * input + outTokens * output + cacheRead * cacheRead + cacheWrite * cacheWrite`.
- **Anthropic semantics:** cache writes are charged at 1.25x input rate AS PART OF input token count, not additive.
- **Captured:** `packages/optimizer/src/pricing.ts` + 7 tests in `optimizer/test/pricing.test.ts`.

### Cascade routing is for subagent model pinning, not tool dispatch
- **Why:** v0.5.0 spec ambiguity. Cascade map at the tool-call level (Haiku for read, Sonnet for edit) doesn't make sense in the LLM worker — all tools run locally, not via LLM dispatch.
- **Decision:** Cascade routing is exposed via the Optimizer for **subagent-level** model pinning (planner/researcher/critic/test-runner = cheap, builder = main). The per-tool cascade map is wired into the LLM call (so the LLM can be hinted about tool cost) but doesn't actually switch models mid-call.
- **Captured:** `packages/optimizer/src/cascade.ts` + `packages/subagent/src/router.ts` (per-subagent model pinning via LlmWorkerOpts.model).

### Tool result cache auto-detects file paths in args
- **Why:** The LLM sends tool calls with `args = { path: "a.ts", ... }`. If callers had to pass `files: ["a.ts"]` explicitly, every tool call would be wrong.
- **Decision:** `ToolResultCache.set` scans args for known file keys (`path`, `file_path`, `file`, `filePath`) and auto-builds the `files` array. Callers can still override via `entry.files`.
- **Captured:** `packages/cache/src/tool-result-cache.ts` + `invalidateForFile(path)` test.

### Optimizer wraps but doesn't break on throw
- **Why:** The optimizer is best-effort optimization. A throw should not crash the agent.
- **Decision:** LlmWorker wraps `optimizer.optimize()` in try/catch; on throw, falls back to raw `provider.complete(messages, opts)`. Tested explicitly.
- **Captured:** `packages/subagent/src/llm-worker.ts` (try/catch around optimize) + test `falls back to raw provider call when optimizer throws`.

### Cost telemetry: cacheRead counts as a HIT, cacheWrite counts as a MISS
- **Why:** Anthropic's semantics — `cache_read_input_tokens` means the model served the request from cached prefix (a hit). `cache_creation_input_tokens` means a fresh write was needed (a miss for the purposes of "how much of this was cached?").
- **Decision:** `CostTracker.recordUsage(usage)` increments `cacheHits++` when `cacheReadTokens > 0`, and `cacheMisses++` when `cacheWriteTokens > 0`.
- **Captured:** `packages/tui-pro/src/cost-display.ts` + test `formats status line with all fields`.

### Repo-map: regex over tree-sitter
- **Why:** tree-sitter requires a native build (WASM download + language packages). Adds significant install complexity for v0.5.0.
- **Decision:** Regex-based symbol scanner (`packages/repo-map/src/extract.ts`) with per-language patterns (TS/JS, Python, Go, Rust, Ruby). Matches the spec's "graceful degrade" path. 1024-token default budget, query-relevance ranking.
- **Future:** v0.7.0 or v0.9.0 can swap in tree-sitter WASM for richer AST info if needed.

### PROMYRA_* env flags default ON; "0" / "false" / "no" / "off" disable
- **Why:** Pi-mono's "YOLO by default" philosophy. Sid prefers low-friction.
- **Decision:** All 5 flags default to true. `apps/pi/src/flags.ts` + `packages/subagent/src/router.ts` consume.
- **Captured:** `apps/pi/src/flags.ts` + 9 tests.

### Bench attribution: 6 flag configs for full cross-coverage
- **Why:** Need to measure each technique's contribution to cost/wall/pass-rate.
- **Decision:** `all-on` (baseline), `all-off` (raw), `cache-off`, `cascade-off`, `parallel-off`, `repomap-off` — 6 configs total. `runAttribution()` runs all and reports deltas vs `all-on`.
- **Captured:** `bench/src/attribution.ts` + 5 tests.

## Housekeeping decisions

- **Why:** Pre-existing across 9 packages. `rootDir: "src"` is incompatible with `include: ["src", "test"]`; errors were hidden by the misconfig.
- **Decision:** Fixed across provider, cache, optimizer, subagent, tools, tasks, checkpoint, memory, skill-bundle.
- **Captured:** This decision log.
