# pi-pro v0.5.0 — Token/Cost Foundation Implementation Plan

> **For agentic workers:** TDD-first, bite-sized tasks, frequent commits. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Cut LLM token cost in half (2x reduction) on the bench while keeping quality at parity and wall time at ≤60% of v0.4.0.

**Architecture:** New packages (`cache`, `optimizer`, `repo-map`) wrap the existing LLM call site. Providers extended with cache-control and cost metadata. TUI gets a live cost status line.

**Tech Stack:** TypeScript, Node 20+, pnpm workspaces, vitest, tree-sitter (lazy-loaded), chokidar (file watcher).

---

## Build order (dependency-driven)

1. **Provider types extension** — `packages/provider/src/types.ts` extended with `cacheControl`, `costUsd`, `cacheReadTokens`, `cacheWriteTokens`. All 3 providers updated to read/write these. *Foundation for everything else.*
2. **`packages/cache`** — `PromptCache` (Anthropic breakpoints + OpenAI prefix) + `ToolResultCache` (LRU + mtime). 100% isolated; TDD-clean.
3. **`packages/optimizer`** — Central decision point. `optimize(turn)` → `OptimizedTurn` with static block, cascade-picked model, cache hints. Cascade router (per-tool hardcoded map).
4. **`packages/repo-map`** — Tree-sitter AST + PageRank. Lazy-load; degrade gracefully.
5. **Wire into `apps/pi`** — Wrap the LLM call site with `optimizer.optimize()`. Parallel tool execution. Cascade dispatch per tool.
6. **TUI telemetry** — `packages/tui-pro` status line + `/cost` command.
7. **Bench** — `bench/run.ts` adds cost, wall, cache-hit columns. Per-flag attribution runs.
8. **Verify** — `pnpm test` (327+), `pnpm bench` (5 tasks × flag configs), regression check, commit.

## Risks (re-stated from spec)

- **Cache API drift** — feature flags gate each technique; degrade to raw LLM call on any failure.
- **Tree-sitter native build** — optional dep; if import fails, repo-map tool returns fallback message.
- **Cascade latency for cheap ops** — bench attribution validates net wall-time is down.
- **Cost telemetry accuracy** — pricing table maintained in `packages/optimizer/src/pricing.ts`; overridable via config.

## Out of scope (re-stated)

LLMLingua · semantic cache · speculative cascades · cross-session memory · TUI polish · sandboxing · nested subagents · plugin marketplace. (All v0.6+.)
