# pi-pro v0.6.0 — Agent Swarm v1 Implementation Plan

> **For agentic workers:** TDD-first, bite-sized tasks, frequent commits. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship an orchestrator + 5-subagent swarm with file scratchpad, per-role worktrees, verification gate, cost cap, and retry-with-feedback. Hit +15pp pass rate over v0.5.0 at ≤1.5x cost and ≤1.8x wall.

**Architecture:** New `packages/swarm` wraps the existing `LlmWorker` from v0.5.0. File-based communication at `.pi-pro/swarm/<id>/`. Per-role git worktrees. Cascade routing applied universally.

**Tech Stack:** TypeScript, Node 20+, pnpm workspaces, vitest, simple-git (for worktree operations), existing v0.5.0 packages (cache, optimizer, repo-map, subagent, tui-pro).

---

## Build order (dependency-driven)

1. **`packages/swarm` skeleton** — package.json, tsconfig, vitest config, types.ts, src/index.ts. No logic yet.
2. **`scratchpad.ts`** with TDD — file-based shared state with atomic writes.
3. **`worktree-pool.ts`** with TDD — git worktree create/destroy, branch management.
4. **`budget.ts`** with TDD — accumulator, soft warn, hard kill, persistence.
5. **`verification-gate.ts`** with TDD — parse test exit code + output.
6. **`types.ts`** with TDD — SwarmId, SwarmPlan, SwarmState, etc.
7. **`plan-writer.ts`** with TDD — markdown plan writer.
8. **`merge.ts`** with TDD — worktree merge back to main.
9. **`optimizer-integration.ts`** with TDD — model selection per subagent.
10. **`orchestrator.ts`** with TDD — state machine, dispatch, verify, retry.
11. **`packages/tui-pro/SwarmPanel.tsx`** with TDD — live swarm status.
12. **`apps/pi/src/commands/swarm.ts` refactor** — wrap with orchestrator.
13. **Verify** — `pnpm -r test`, typecheck, build, all green.

### Task 10: `orchestrator.ts` with TDD

**Files:**
- Create: `packages/swarm/src/orchestrator.ts`
- Create: `packages/swarm/test/orchestrator.test.ts`

**Tests (heavy — orchestrator is complex):**
- `Orchestrator.run(goal, opts)` — full state machine
- Phase 1: PLAN. Writes plan.md.
- Phase 2-3: RESEARCH + PLAN2 run in parallel.
- Phase 4: BUILD with retry policy. On pass, advance. On fail (3 attempts), surface error.
- Phase 5: TEST. If fail, retry BUILD with test output as feedback.
- Phase 6: CRITIQUE. If critique finds issues, retry BUILD with critique as feedback.
- Phase 7: DECIDE. Test passed? → advance. Else → retry.
- Phase 8: MERGE. If dry-run, skip.
- `Orchestrator.run` returns SwarmResult
- State machine: state.json persisted between phases (resume works)
- Budget check before each subagent dispatch (if killed, halt with `paused: budget exceeded`)
- Worktree creation before builder (if fails, surface error)
- Multica bypass: not applicable here (orchestrator only)

**Implementation:**
- `Orchestrator` class
- Phase methods: `phasePlan`, `phaseResearchAndPlan`, `phaseBuild`, `phaseTest`, `phaseCritique`, `phaseDecide`, `phaseMerge`
- Each phase: read inputs from scratchpad, call LlmWorker, write outputs, update state.json
- Retry: re-call with previous evidence as additional context

- [ ] Write tests (use mock LlmWorker)
- [ ] Write `src/orchestrator.ts`
