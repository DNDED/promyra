## 1. Goal

On the 5-task LLM bench (`bench/tasks/`), on the same model, vs. v0.4.0:

| Metric | Target | Measurement |
|---|---|---|
| **Cost** (input tokens avg) | ≤ 50% of v0.4.0 | `bench/run.ts` total in-tokens / 5 tasks |
| **Quality** (pass rate) | ≥ 95% of v0.4.0 (5% absolute variance) | bench pass count |
| **Wall** (avg task time) | ≤ 60% of v0.4.0 (~18s) | bench wall clock / 5 tasks |
| **Regression** | 327 existing tests pass | `pnpm test` |
| **Cache hit rate** | ≥ 60% on warm sessions (turn 3+) | per-turn telemetry |

## 2. Non-Goals

LLMLingua compression, semantic response cache, speculative cascades, cross-session memory, TUI polish, OS-level sandboxing, nested subagents, plugin marketplace. (All parked for v0.6+ per roadmap.)

## 4. Components

### 4.1 `packages/cache` (new)
- **`PromptCache`** — per-provider cache-breakpoint manager
  - Anthropic: inserts `cache_control: ephemeral` on system block, repo-map block, tool-definitions block (4 max breakpoints)
  - OpenAI: orders message array so static prefix is first; sets `prompt_cache_key` for shared prefixes
  - opencode-go: pass-through (no cache control)
  - Tracks hits/misses per session
- **`ToolResultCache`** — in-memory LRU, max 256 entries
  - Key: `sha256(tool_name + normalized_args)`
  - Invalidation: file-mtime check on read/grep/glob; explicit invalidation on edit/write
  - TTL: session lifetime
  - Cached: `read`, `grep`, `glob`, `git status`, `git diff`

### 4.2 `packages/repo-map` (new)
- **`get_repo_map(query, k)`** — Aider-style tree-sitter based
  - Parses all files in current worktree → AST graph
  - PageRank-style ranking of symbols by relevance to `query`
  - Returns top-k symbols (default 1024 tokens, configurable)
  - Lazy build: only on first call per session per worktree
  - Refresh: file watcher (chokidar) on `.pi-pro/`, `*.ts`, `*.js`, `*.py`, etc.
- **`get_symbol(file, name)`** — tree-sitter symbol lookup
- **`find_references(symbol)`** — call-graph traversal
- **Graceful degrade:** parse errors on individual files skip those files; never break the map

### 4.3 `packages/optimizer` (new, central decision point)
- **`optimize(turn: TurnContext): OptimizedTurn`**
  - Computes static block once: system prompt + repo map + tool definitions (cached)
  - Computes dynamic block: user message + recent tool results (un-cached)
  - Marks cache breakpoints (Anthropic) or orders for prefix caching (OpenAI)
  - Picks model per tool call (cascade routing — see §5)
  - Returns OptimizedTurn for LLM dispatch
- **Failure mode:** throws → caller falls back to raw LLM call (no optimization). Never blocks the agent.

### 4.4 Cascade Router (in `packages/optimizer`)
Per-tool hardcoded mapping. No classifier model; no router overhead.

| Operation | Cascade Model | Rationale |
|---|---|---|
| `grep`, `glob` | **Cheap** (see §5.1) | Read-only, structured output |
| `git status`, `git diff` | **Cheap** | Deterministic |
| `read` (single file, not in edit flow) | **Cheap** | Reading is cheap to verify |
| `edit`, `write`, `apply_patch` | **Main** | High-stakes, precision required |
| `bash` (test runners, scripts) | **Main** | Output interpretation matters |
| `bash` (install, format, lint) | **Cheap** | Deterministic, low-stakes |
| Subagent `explore`, `web` | **Cheap** | Already pinned cheap in v0.4 |

**5.1 Cheap model selection:** resolved per provider, in priority order:
- Anthropic → `claude-haiku-4-5`
- OpenAI → `gpt-5-mini`
- opencode-go → `deepseek-v4-flash`

Override per LLM call: `{ cascade: { tool, model } }`. Fallback to main on error (logged once per session).

### 4.5 `packages/provider` (modified)
- Add cache-control per provider (Anthropic breakpoints, OpenAI prefix, opencode-go passthrough)
- Return cost metadata in every response: `{ input_tokens, output_tokens, cache_read_tokens, cache_write_tokens, cost_usd }`
- Cost pricing table per model (loaded from config; can be overridden)

### 4.6 `apps/pi` (modified — was `packages/agent` in spec; actually lives in `apps/pi/src/commands/`)
- Wrap LLM call site with `optimizer.optimize(turn)`
- Collect all `tool_use` blocks from one assistant turn; execute concurrently (Promise.all on independent tool calls)
- Dispatch each tool call with cascade-resolved model
- Tool result cache check before execution

### 4.7 `packages/tui-pro` (modified)
- **Status line** (always visible bottom row): `tok:1.4k in / 380 out | $0.012 | cache:74% | 12:34:56`
- **`/cost` slash command** — breakdown by category (system, tools, files, conversation) + per-task totals + session total
- **Per-turn cost** inline next to assistant message (small, subtle)
- **No network calls** for telemetry; all local

### 4.8 `bench/run.ts` (modified)
- New columns: `cost_usd`, `cache_hit_pct`, `wall_seconds`
- Per-technique attribution: bench with each flag off individually to isolate each technique's contribution
- Output JSONL includes raw per-turn token/cost data

## 5. Feature Flags

```bash
PROMYRA_CACHE=0          # disable prompt cache
PROMYRA_REPO_MAP=0       # disable repo map (block + tool)
PROMYRA_CASCADE=0        # disable cascade routing
PROMYRA_PARALLEL_TOOLS=0 # disable parallel tool execution
PROMYRA_TELEMETRY=0      # disable cost telemetry
```

All default ON. Used for bench attribution and gradual rollout.

## 6. Error Handling

| Failure | Behavior |
|---|---|
| Cache miss on turn 1 | No savings; warms from turn 2 |
| Cascade model unavailable | Fall back to main; log warning once per session |
| Tree-sitter parse error on file | Skip file; log; continue with rest of map |
| Tool result cache stale (mtime mismatch) | Re-execute; cache new result |
| Cost telemetry fails | Log error; continue without telemetry |
| Repo map exceeds 1k tokens | Rank by PageRank, truncate, log warning |
| Optimizer throws | Fall back to raw LLM call (no optimization) |
| tree-sitter native build fails | Lazy-load; fall back to grep+glob for repo map |

## 8. Risks & Mitigations

1. **Provider cache-control API drift** — APIs are stable but third-party (opencode-go) is passthrough. *Mitigation:* feature flag, graceful fallback.
2. **Tree-sitter native build** — new dep. *Mitigation:* optional, lazy-loaded; degrades to grep+glob.
3. **Cascade adds latency for cheap ops** — net wall-time still down because cache + parallel dominate. *Mitigation:* bench attribution shows per-flag impact.
4. **Telemetry data leakage** — local-only, no network. *Mitigation:* audit logs; no PII in cost data.
5. **Repo map quality on large codebases** — 1k token budget may be insufficient. *Mitigation:* `get_repo_map` tool for on-demand refinement; configurable budget.
